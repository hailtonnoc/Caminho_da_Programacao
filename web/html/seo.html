Conceitos

O que é?

Search Engine Optimization é uma metodologia de otimização das caraterísticas estruturais do site para adequação ao sistema de busca (geralmente o google). Este sistema é retro alimentado pelo tipo de experiência que o usuário pode ter.
Para isso o sistema analisa o site compilando tudo em uma Palavra chave. consequentemente esta deve ser proxima a pesquisa direta do usuário.

Como utilizar?

  1. Analise do negocio e stakeholders.
  2. Escolhendo corretamente a palavra chave para o negocio.
  3. Planejamento do conteúdo.
  4. Escrevendo o HTML coerente (o que convenhamos, é o mínimo).
Indicar bem o conteúdo nos títulos h1, h2, h3... colocar da maneira que o usuário pesquisa.

Analise de publico alvo stakeholders

  • Quem vai acessar o site? (pensar em como o negocio seria fechado fisicamente)
  • Genero
  • Qual a localidade da prestação? pode ser util colocar o nome do local para espeficar.

Algumas ferramentas para analise de palavras chave:

  • Google Trends
  • Ubersuggest
  • MOZ
  • Google Ads
  • Ahrefs
  • SemRush

Ferramentas para planejamento de conteúdo

  • Jornada de compra experiência do cliente físico (entender o processo de compra)
  • Identifica o grupo alvo util para saber o tipo de quem acessa
  • Pesquisas relacionadas entender o que cerca a pesquisa

Ferramenta para desempenho

  • Verifica o desempenho do site

Ferramentas extras

  • Google Analytics, Navegg para analise de interessados, localização, demografia...
  • Frankmarcel para contar caracteres, lembrando que ideal é 60 e no maximo 90 e para descrições 180 a 240.
  • Tag Manager (talvez procurar um curso)

Planejamento de Conteudo

  1. Escolher a palavra chave
  2. Analisar em ferramenta e verificar os concorrentes
    • Verificar dominios Ferramenta verificar dominio
    • Tempo (quanto mais tempo mais propriedade)
    • Servidor (qualidade pois tempo de carregamento conta)
    • Titulo do site x palavra chave exata
    • Analise de estrutura do site atraves do web developer, document outline e builtwith

Incorporando sitemap

Sitemap é a uma relação de URL's em XML e ajuda na relevância e compreensão do site como um todo.
Normalmente o sitemap se faz necessário para arquiteturas maiores de 500 paginas.
O mesmo é criado em um arquivo XML e é adicionado a pasta root do site. Ele se faz extremamente importante em e-commerce uma vez que sua atualização garante a indexação da pagina do produto no google.
Mantenha o sitemap atualizado, atenção ao excluir e ao adicionar paginas. inicialmente pode usar o Gerador de site map é interessante buscar uma cronjob.
Obs: Não esqueça de notificar ao Google Search Console.

URL Amigável

Para aumentar a relevância no Search é interessante ter url's amigáveis, sem códigos, ID's, numerações ou extensões. verificar em pesquisa a parte
